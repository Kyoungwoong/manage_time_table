# ⏰ 타임테이블 관리 서비스 기획서 (v1)

## 1. 서비스 개요

### 서비스 이름 (가칭)
- TimeWeave
- Slot
- Dayflow
- Timi

> 이름은 개발하면서 vibe 맞춰 최종 결정

### 한 줄 소개
> **하루의 시간을 시각적으로 설계하고, 반복되는 일정 관리의 부담을 줄여주는 타임테이블 서비스**

---

## 2. 문제 정의 (Why)

### 사용자가 겪는 문제
- 기존 캘린더는 **이벤트 중심**이라 하루 구조를 파악하기 어려움
- 공부·운동·루틴처럼 **시간 블록 기반 일정 관리**가 불편함
- 여러 도구(Notion, 캘린더, 메모)를 오가며 관리함
- 계획 대비 실제 사용 시간을 직관적으로 인지하기 어려움

### 핵심 Pain Point
- 하루의 흐름이 한눈에 보이지 않음
- 반복 일정 관리가 번거로움
- 회고가 감정적으로 남지 않고 데이터로만 남음

---

## 3. 타겟 사용자

### 1차 타겟 (MVP)
- 대학생 / 수험생
- 자기관리, 루틴 관리에 관심 있는 사용자
- 하루를 “시간 블록”으로 관리하고 싶은 사람

### 2차 타겟 (확장)
- 프리랜서 / 개발자 / 크리에이터
- 운동, 공부 루틴을 지속적으로 관리하는 사용자

---

## 4. 핵심 컨셉

> **하루를 이벤트가 아닌 ‘시간의 흐름’으로 관리한다**

- 하루 = 타임라인
- 일정 = 시간 블록
- 계획은 고정되지 않고 언제든 수정 가능
- 시각적으로 직관적이고 부담 없는 사용 경험

---

## 5. 핵심 기능 (MVP)

### 5.1 하루 타임테이블 뷰
- 00:00 ~ 24:00 세로 타임라인
- 드래그로 시간 블록 생성 및 조정
- 색상/아이콘으로 일정 유형 구분

예시:
- 🟦 공부
- 🟩 운동
- 🟨 휴식

---

### 5.2 반복 타임테이블
- 요일별 패턴 설정 (평일 / 주말 등)
- 반복 패턴을 날짜에 자동 적용
- 특정 날짜 예외 처리 가능

---

### 5.3 실제 사용 기록
- 각 시간 블록에 상태 표시
    - 완료
    - 일부 수행
    - 미수행
- 정확한 타이머 대신 **체크 기반 기록**
- 부담 없는 회고 목적

---

### 5.4 하루 회고 요약
- 계획 대비 실제 사용 비율 시각화
- 하루 전체 사용 흐름 요약
- 한 줄 메모 작성 가능 (선택)

---

## 6. 차별점

| 항목 | 일반 캘린더 | 타임테이블 서비스 |
|---|---|---|
| 중심 | 이벤트 | 시간 흐름 |
| 관리 단위 | 일정 | 시간 블록 |
| 반복 관리 | 불편 | 패턴 기반 |
| 회고 | 제한적 | 시각적 요약 |
| 감성 | 딱딱함 | 부드럽고 직관적 |

---

## 7. UX / UI 방향

### 톤 & 무드
- 미니멀
- 파스텔 또는 muted 컬러
- 부드러운 애니메이션

### UX 원칙
- 클릭 수 최소화
- 만드는 재미 강조
- 언제든 수정 가능, 실패 부담 없음

---

## 8. 기술적 MVP 범위

### 포함
- 사용자 인증 (이메일 또는 소셜)
- 하루 타임테이블 CRUD
- 반복 패턴 저장
- 기본 통계/요약

### 제외 (초기 범위 아님)
- 알림 기능
- 협업 기능
- 정밀 타이머 기능

---

## 9. 확장 아이디어 (After MVP)
- 주간 / 월간 패턴 분석
- 일정 패턴 추천
- 집중 시간 분석
- 모바일 앱
- 외부 서비스 연동 (캘린더, Notion 등)

---

## 10. 성공 지표 (초기)
- 하루 타임테이블 생성 비율
- 연속 사용 일수
- 반복 패턴 생성 비율
- 사용자 주관적 만족도 (“하루가 정리됐다”는 피드백)

---

## 11. 개발 순서 제안

1. 하루 타임라인 UI 구현
2. 시간 블록 생성 / 수정
3. 데이터 저장 및 로딩
4. 반복 패턴 기능
5. 간단한 회고/요약 기능
